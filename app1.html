Warehouse Management System(Click enter APP)

I will introduce the construction ideas and technology stack of the warehouse management system in detail, hoping to give a good impression on the interviewer.

The front end of the project uses the React framework based on Javascript, mainly using react class to create components, using D3.js to build visual charts, using ‘this.state’ for state management, using Axios to request API to obtain back-end responses, and using life cycle functions to implement components Life cycle management. At the same time, I also have the ability to use react hook to create components and use react hook or redux for state management, which will be demonstrated in my next sample application-Study Notes System. Due to the eagerness to find a new job, UI design It's lacking, which means he may look a little ugly, please forgive me.

The backend of the project is based on NodeJS, uses Express.js as the framework, and reads, writes and queries data through middleware. The project does not have a separate database. In order to improve the development progress, the data will be written directly into the JSON file.

The project has been deployed on E2C server,using Nginx. Due to the limited access, I did not create an ELB for load balancing. At the same time, I also successfully used S3+Beanstalk to complete the deployment of the project

Entering the project, the first page is the login interface. There are two user modes in the system, one is administrator mode and the other is common user mode. Users can create common users by registering and have a built-in admin user: admin@gmail.com/Aa123456!

You can use the above email address and password to log in by administrator mode. Administrator mode can manage the permissions of other users after logging in to the system.

When logging in, use Axios to send a Post request to the backend. After verified, the key will be sent to the browser and stored in a cookie, and the login status will be maintained in this way. The next time you log in, the key will be verified first, and if the key is verified, you will log in directly

If you lack an account, you will be taken to the registration page, where you need to register as required. The filled content will be verified, and if the verification fails, a prompt will be given as shown in the figure below

The Category page, you can add, delete, and modify the types of inventory products to facilitate classification management. You can view subcategories by clicking View subcategory

The Commodity page, you can add, modify, and delete specific product information, change the sales status of the goods through the Sale state, and complete the keyword search through the search box.

When the user clicks Detail, you will enter the product details page, where the user can change and view more product information

Entering the Chart page, you can view the responsive chart created with D3.js. This example shows the current stock quantity of all products. I didn't do much diagramming. Frankly speaking, in my past work, it was also inseparable to find development documents and use Google search. We can complete the development by modifying the parameters of beautiful charts in a shorter time. However, I need to understand the concept of his canvas, SVG, etc.

The Role page, the administrator account can view and modify other user permissions. For example, when the Chart permission of the junzheng.liu@empeal.com user is not enabled, he will not be able to use the Chart function anymore.
